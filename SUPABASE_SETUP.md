# Supabase Integration Setup

The Careers page is now integrated with Supabase for dynamic job listings and application submissions.

## Required Supabase Tables

### 1. `job_listings` Table

Create this table in your Supabase database:

```sql
create table job_listings (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()),
  title text not null,
  location text not null,
  salary text not null,
  type text not null,
  description text not null,
  requirements text[] not null default '{}',
  is_active boolean not null default true
);

-- Enable Row Level Security
alter table job_listings enable row level security;

-- Allow public read access
create policy "Allow public read access" on job_listings
  for select to anon using (true);

-- Allow authenticated users to insert/update (for admin purposes)
create policy "Allow authenticated insert" on job_listings
  for insert to authenticated with check (true);

create policy "Allow authenticated update" on job_listings
  for update to authenticated using (true);
```

### 2. `job_applications` Table

Create this table for storing applications:

```sql
create table job_applications (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()),
  name text not null,
  email text not null,
  phone text,
  position text not null,
  country text not null,
  english_level text not null,
  experience text not null,
  message text,
  resume_url text
);

-- Enable Row Level Security
alter table job_applications enable row level security;

-- Allow public insert (for applications)
create policy "Allow public insert" on job_applications
  for insert to anon with check (true);

-- Only allow authenticated users to read (for admin purposes)
create policy "Allow authenticated read" on job_applications
  for select to authenticated using (true);
```

### 3. Storage Bucket for Resumes

Create a storage bucket named `applications` for storing resume files:

1. Go to Storage in Supabase Dashboard
2. Create a new bucket called `applications`
3. Set the bucket to public (or configure appropriate policies)
4. Add these policies:

```sql
-- Allow public uploads to applications bucket
create policy "Allow public uploads" on storage.objects
  for insert to anon with check (bucket_id = 'applications');

-- Allow public to read files
create policy "Allow public read" on storage.objects
  for select to anon using (bucket_id = 'applications');
```

## Environment Variables

Add these to your `.env` file:

```
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

Get these values from your Supabase project:
- Project Settings → API → Project URL
- Project Settings → API → anon/public key

## Fallback Behavior

If Supabase is not configured (missing env vars), the careers page will:
1. Display hardcoded fallback job listings
2. Submit applications via FormSubmit.co (the original method)

Once you add the Supabase credentials, it will automatically switch to using the database.

## Testing

1. Add your Supabase credentials to `.env`
2. Run `npm run dev`
3. Check that job listings load from your database
4. Submit a test application
5. Verify the application appears in your Supabase table
